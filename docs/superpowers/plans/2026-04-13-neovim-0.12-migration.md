# NeoVim 0.12.1 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the NeoVim dotfiles config to fully leverage NeoVim 0.12.1 — fix deprecated APIs, adopt new built-in LSP features, and migrate nvim-treesitter to the modern `main` branch API.

**Architecture:** Changes are organized from simplest/safest to most complex. Quick deprecation fixes first, then global options, then LSP feature enablement, and finally the large treesitter rewrite. Each task is independently verifiable via headless checks.

**Tech Stack:** NeoVim 0.12.1, Lua, lazy.nvim, nvim-treesitter (main branch), nvim-lspconfig, none-ls.nvim

**Design Spec:** `docs/superpowers/specs/2026-04-13-neovim-0.12-migration-design.md`
