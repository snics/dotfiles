# NeoVim 0.12.1 Migration Design

Migration of the NeoVim dotfiles configuration from a pre-0.12 state to fully leverage
NeoVim 0.12.1 features, fix deprecated APIs, and adopt new built-in capabilities.

## Current State

- NeoVim v0.12.1 installed
- Plugin manager: lazy.nvim (stays)
- LSP setup already uses modern `vim.lsp.config()` / `vim.lsp.enable()` API
- nvim-treesitter still on `master` branch (frozen, deprecated API)
- Several deprecated Lua APIs still in use
- New 0.12 LSP features (CodeLens, onTypeFormatting, linked editing, inlay hints) not configured

## Scope

### 1. Critical: nvim-treesitter Migration to `main` Branch

**Problem:** `require("nvim-treesitter.configs").setup()` is deprecated. The `master` branch
is locked and maintained only for backward compatibility with NeoVim 0.11. While it may
still work on 0.12, the `main` branch is the actively developed version with the modern API.

**Important:** The `main` branch README states the plugin does **not support lazy-loading**.
The current config lazy-loads via `event = { "BufReadPre", "BufNewFile" }` — this must be
removed. Set `lazy = false` instead.

**Changes to `treesitter.lua`:**

Core treesitter spec:
- Set `lazy = false` (remove `event` trigger — plugin does not support lazy-loading)
- Remove `require("nvim-treesitter.configs").setup()` call
- Use `vim.treesitter.start()` in a `FileType` autocmd for highlighting
- Set `vim.bo.indentexpr` manually for treesitter-based indentation
- Replace `ensure_installed` with `require("nvim-treesitter").install()` call
- Replace `auto_install = true` with equivalent new-API approach
- Remove `incremental_selection` block entirely (replaced by native — see section 8)
- Switch branch from `master` to `main` in lazy spec

### 2. Critical: nvim-treesitter-textobjects Migration to `main` Branch

**Problem:** Plugin's `main` branch has a new API — textobjects are no longer configured
via `require("nvim-treesitter.configs").setup({ textobjects = ... })`.

**Changes to `treesitter.lua` (textobjects spec):**

- Replace config-table approach with `require("nvim-treesitter-textobjects").setup()`
- Convert all `select` keymaps to explicit `vim.keymap.set()` calls using
  `require("nvim-treesitter-textobjects.select").select_textobject()`
- Convert all `move` keymaps to explicit `vim.keymap.set()` calls using
  `require("nvim-treesitter-textobjects.move").goto_next_start()` etc.
- Convert all `swap` keymaps to explicit `vim.keymap.set()` calls using
  `require("nvim-treesitter-textobjects.swap").swap_next()` etc.
- Preserve ALL existing textobject keymaps (`af`, `if`, `ac`, `ic`, `aa`, `ia`,
  `ai`, `ii`, `al`, `il`, `ab`, `ib`, `aC`, `iC`, `as`, `is`, `ak`, `av`,
  `at`, `it`, `aS`, `]m`, `[m`, `]c`, `[c`, etc.)
- Preserve `selection_modes` configuration
- Switch branch from `master` to `main` in lazy spec
- Remap `an` from `@number.inner` to `aN` (free up `an` for native incremental selection)

### 3. High: Fix Deprecated `client.supports_method()` Dot-Syntax

**Problem:** `client.supports_method()` (dot syntax) is deprecated since 0.11 in favor
of `client:supports_method()` (colon syntax — proper Lua method call with implicit `self`).

**File:** `plugins/none-ls.lua` (line 265)

Change:
```lua
-- Old: if client.supports_method("textDocument/formatting") then
-- New: if client:supports_method("textDocument/formatting") then
```

Note: The new LSP feature calls in this spec (CodeLens, linked editing, etc.) already
use colon syntax correctly.

### 3b. High: Fix Deprecated `vim.diagnostic.goto_next/prev`

**Problem:** `vim.diagnostic.goto_next()` and `vim.diagnostic.goto_prev()` are deprecated
since 0.11 in favor of `vim.diagnostic.jump()`.

**Files affected:**

`config/keymaps.lua` (lines 59-66):
- `]e` → `vim.diagnostic.jump({ count = 1, severity = vim.diagnostic.severity.ERROR })`
- `[e` → `vim.diagnostic.jump({ count = -1, severity = vim.diagnostic.severity.ERROR })`
- `]w` → `vim.diagnostic.jump({ count = 1, severity = vim.diagnostic.severity.WARN })`
- `[w` → `vim.diagnostic.jump({ count = -1, severity = vim.diagnostic.severity.WARN })`

`plugins/lspconfig.lua` (lines 24-25):
- `[d` → `vim.diagnostic.jump({ count = -1 })`
- `]d` → `vim.diagnostic.jump({ count = 1 })`

### 4. Low: DAP Sign Definitions (No Change Needed)

**Clarification:** The 0.12 breaking change for `sign_define()` applies specifically to
**diagnostic signs** — not all signs globally. DAP breakpoint signs still use
`vim.fn.sign_define()` and this is still the supported approach in nvim-dap.

**File:** `plugins/dap.lua` (lines 158-162) — **no changes needed**.

### 5. Medium: Global `winborder` Setting

**New in 0.11.** Sets consistent borders for all floating windows globally.

**File:** `config/options.lua`

Add: `vim.o.winborder = "rounded"`

This makes per-plugin border configuration redundant. After setting:
- nvim-cmp bordered windows (lines 165-172 of `nvim-cmp.lua`) can be simplified
- DAP floating border config becomes redundant
- All hover docs, diagnostic floats, etc. get consistent rounded borders

### 6. Low: Remove Redundant `termguicolors`

**File:** `config/options.lua` (line 27)

Remove `opt.termguicolors = true` — auto-enabled since 0.10 and fully
redundant in 0.12. The comment already notes this.

### 7. Configure: Inlay Hints for Rust/Go/TypeScript/JavaScript

**Feature:** Inline type annotations and parameter names shown by LSP.
Available since NeoVim 0.10 via `vim.lsp.inlay_hint.enable()`, but not yet
configured in this setup. The API is stable in 0.12.

**Server support:**
- rust-analyzer: Supported (good defaults, no config changes needed)
- gopls: Supported but **all hints are OFF by default** — needs settings
- ts_ls: Supported but **all hints are OFF by default** — needs settings
- cssls, yamlls: Not supported (no action)

**Changes:**

`plugins/lspconfig.lua` — in the `LspAttach` callback, add:
```lua
vim.lsp.inlay_hint.enable(true, { bufnr = ev.buf })
```
(Only for buffers with LSP clients that support it. The API already handles
capability checking internally.)

`config/lsp/servers/gopls.lua` — add hint settings:
```lua
hints = {
    assignVariableTypes = true,
    compositeLiteralFields = true,
    compositeLiteralTypes = true,
    constantValues = true,
    functionTypeParameters = true,
    parameterNames = true,
    rangeVariableTypes = true,
}
```

New file `config/lsp/servers/ts_ls.lua` — create with inlay hint settings:
```lua
settings = {
    typescript = {
        inlayHints = {
            includeInlayParameterNameHints = "all",
            includeInlayVariableTypeHints = true,
            includeInlayFunctionLikeReturnTypeHints = true,
            includeInlayPropertyDeclarationTypeHints = true,
            includeInlayFunctionParameterTypeHints = true,
            includeInlayEnumMemberValueHints = true,
        },
    },
    javascript = {
        inlayHints = {
            includeInlayParameterNameHints = "all",
            includeInlayVariableTypeHints = true,
            includeInlayFunctionLikeReturnTypeHints = true,
            includeInlayPropertyDeclarationTypeHints = true,
            includeInlayFunctionParameterTypeHints = true,
            includeInlayEnumMemberValueHints = true,
        },
    },
}
```

Existing `<leader>uh` toggle via Snacks continues to work for on/off.

### 8. New: Native Treesitter Incremental Selection

**Feature:** Built-in `an`/`in` (expand/shrink) and `]n`/`[n` (siblings) in visual
mode (`x` mode). These are default mappings added in 0.12.

**Changes:**
- Remove `incremental_selection` block from treesitter.lua core spec
- Remap `an` textobject (`@number.inner`) to `aN` to free up `an` for native use
- Native `an`/`in`/`]n`/`[n` are active by default — no config needed
- Optionally add `<C-Enter>`/`<C-Backspace>` as aliases in visual mode for
  muscle memory (calling `vim.treesitter._select.select_parent/select_child`)

### 9. Configure: LSP CodeLens (Reimplemented in 0.12)

**Feature:** Virtual lines above functions showing "Run Test", "Debug",
"Implementations" etc. CodeLens existed before 0.12, but 0.12 reimplemented the
handler with virtual lines display and added `vim.lsp.codelens.enable()` API.

**Server support:**
- rust-analyzer: Supported (defaults are good — Run/Debug/Implementations ON)
- gopls: Supported (test lens OFF by default, enable it)
- ts_ls: Supported (References/Implementations)
- yamlls, cssls, jsonls: Not supported

**Changes:**

`plugins/lspconfig.lua` — in `LspAttach`, enable CodeLens:
```lua
if client and client:supports_method("textDocument/codeLens") then
    vim.lsp.codelens.enable(true, { bufnr = ev.buf })
end
```

`config/lsp/servers/gopls.lua` — add codelens test setting:
```lua
codelenses = {
    test = true,  -- show "Run Test" on Go test functions
}
```

Default mapping `grx` already exists for running CodeLens actions.

### 10. New: LSP onTypeFormatting (Rust Only)

**Feature:** Auto-formatting while typing trigger characters.

Only enabled for rust-analyzer (smart semicolons after `=`, method chain indent
after `.`). Disabled for Go (gofmt-on-save is sufficient) and TypeScript
(conflict with prettierd/biome via none-ls).

**Changes:**

`plugins/lspconfig.lua` — in `LspAttach`:
```lua
if client and client.name == "rust_analyzer" then
    vim.lsp.on_type_formatting.enable(true, { client_id = client.id })
end
```

### 11. New: LSP Linked Editing Range + nvim-ts-autotag Adjustment

**Feature:** Synchronized tag renaming (edit opening tag, closing tag follows).

**Split responsibilities:**
- nvim-ts-autotag: Keeps auto-close (`enable_close = true`)
- nvim-ts-autotag: Gives up auto-rename (`enable_rename = false`)
- Native linked editing range: Handles auto-rename (LSP-accurate)

**Changes:**

`treesitter.lua` — update nvim-ts-autotag config:
```lua
opts = {
    opts = {
        enable_close = true,
        enable_rename = false,  -- changed: native linked editing handles this
        enable_close_on_slash = false,
    },
},
```

`plugins/lspconfig.lua` — in `LspAttach`:
```lua
if client and client:supports_method("textDocument/linkedEditingRange") then
    vim.lsp.linked_editing_range.enable(true, { client_id = client.id })
end
```

### 12. New: `:Undotree` Keybinding

**Feature:** Built-in visual undo tree navigator (side panel with ASCII graph).

Requires `:packadd nvim.undotree` before first use. Complements existing
`<leader>su` (Snacks undo picker with diff preview and search).

**Changes:**

`plugins/snacks.lua` or `config/keymaps.lua` — add keybinding:
```lua
vim.keymap.set("n", "<leader>sU", function()
    vim.cmd("packadd nvim.undotree")
    vim.cmd("Undotree")
end, { desc = "Undo Tree (native)" })
```

Update which-key if needed.

### 13. New: `:restart` Keybinding

**Feature:** Restart NeoVim without closing the terminal, all UIs reconnect.

**Changes:**

`config/keymaps.lua` — add:
```lua
vim.keymap.set("n", "<leader>qr", "<cmd>restart<CR>", { desc = "Restart NeoVim" })
```

Fits in existing `<leader>q` group ("Quit").

## Plugins Kept

All existing plugins remain. No plugins are removed. Changes are:
- nvim-ts-autotag: `enable_rename` set to `false` (native handles it)
- nvim-treesitter: Branch switch `master` → `main`, API migration
- nvim-treesitter-textobjects: Branch switch, API migration

## Plugins NOT Added

- blink.cmp (user decided against)
- mini.icons (staying with nvim-web-devicons)

## Automated Verification (Headless)

Run after each implementation step to catch regressions. All commands run
headless (no UI needed) and output pass/fail to stdout.

### Check 1: Lua Syntax Validation
Verify all 66+ config files parse without syntax errors:
```bash
nvim --headless -c 'lua local errors = {}; local files = vim.fn.glob(vim.fn.stdpath("config") .. "/lua/**/*.lua", false, true); for _, f in ipairs(files) do local ok, err = loadfile(f); if not ok then table.insert(errors, f .. ": " .. err) end end; if #errors > 0 then for _, e in ipairs(errors) do print("ERROR: " .. e) end; os.exit(1) else print("OK: All " .. #files .. " Lua files parse successfully") end; os.exit(0)'
```

### Check 2: Plugin Load Errors
Verify all lazy.nvim plugins load without errors:
```bash
nvim --headless -c 'lua vim.defer_fn(function() local errors = {}; for name, plugin in pairs(require("lazy.core.config").plugins) do if plugin._.has_errors then table.insert(errors, name .. ": load error") end end; if #errors > 0 then for _, e in ipairs(errors) do print("ERROR: " .. e) end; os.exit(1) else print("OK: All plugins loaded without errors") end; os.exit(0) end, 3000)'
```

### Check 3: Startup Messages
Verify no errors or warnings during startup:
```bash
nvim --headless -c 'lua vim.defer_fn(function() local msgs = vim.api.nvim_exec2("messages", {output=true}).output; if msgs and msgs:match("[Ee]rror") then print("ERRORS FOUND:\n" .. msgs); os.exit(1) elseif msgs and msgs:match("[Ww]arn") then print("WARNINGS FOUND:\n" .. msgs) else print("OK: No errors or warnings in startup messages") end; os.exit(0) end, 3000)'
```

### Execution Protocol

During implementation, run Checks 1-4 after **every file change**. Check 5
after all changes are complete. Any new ERROR not in the pre-existing list
above means a regression was introduced.
