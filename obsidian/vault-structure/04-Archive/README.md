# 📦 Archive

**Purpose:** Completed projects and inactive items for future reference.

## What goes in Archive?

- **Completed Projects** - Finished with outcomes documented
- **Inactive Areas** - Former responsibilities no longer relevant
- **Old Resources** - Outdated but worth keeping
- **Past Experiments** - Things you tried that didn't work out

## Archive Structure

```
04-Archive/
├── Projects/          # Completed projects (by year)
│   ├── 2025/
│   ├── 2024/
│   └── 2023/
├── Areas/             # Former areas of responsibility
├── Resources/         # Outdated reference materials
└── Experiments/       # Failed experiments, learnings
```

## When to Archive

- Project is completed (document learnings first!)
- Area is no longer relevant
- Information is outdated but worth keeping
- You want to clean up active workspace

## Archive Best Practices

1. **Add outcome summary** before archiving projects
2. **Extract learnings** - what worked, what didn't
3. **Keep it searchable** - maintain good naming and tags
4. **Review annually** - delete truly irrelevant items
5. **Link from active notes** - can still reference archived content

## Retrieval

Archived items are still searchable and linkable:
- Use global search to find archived content
- Graph view shows connections
- Backlinks work across all folders

---

*Archive is not a graveyard - it's a curated library of past work.*
